I Подготовка:
Архив GeoScripts распаковать, распакованную папку GeoScripts вместе с файлами:
GeoAGR.mse
GeoAGR.ini
GeoAGR.exe
и всеми остальными папками и файлами положить в корень вашего 3ds max. Например,
C:\Program Files\Autodesk\3ds Max 2024\GeoScripts\
C:\Program Files\Autodesk\3ds Max 2025\GeoScripts\

Возможно понадобится:
-Внести в исключения для Windows Defender папку GeoScripts и все программы (exe) в ней.
-Изменить UAC для доступа к папке GeoScripts.
-Изменить права доступа у текущего пользователя к папке GeoScripts и файлам в них, на чтение, на создание и запись новых.
-Возможно потребуется помощь вашего системного администратора.

Для архивации файлов скриптом, скачать и установить 7z.exe. 
Cсылка на установщик: https://www.7-zip.org/download.html
Путь пока в скрипт вшит такой:
C:\Program Files\7-Zip\7z.exe

 
II Установка скрипта в 3ds max производится также как любого макроскрипта: 
1. Перетащить файл GeoAGR_start.ms в окно проекции запущенного 3ds max. Ничего визуального не произойдет. 
2. Перейти в настройки интерфейса: Меню сверху -> Customize -> Customize User Interface. 
3. Выбрать закладку Toolbars. 
4. Выбрать в списке слева категорию Geoscripts и выбрать скрипт GeoAGR в списке справа, перетащить его на панель в удобное для вас место.
5. Вызвать скрипт GeoAGR из панели.

III Запуск скрипта:
1. При первом запуске скрипта из панели (или меню), запустится форма ввода данных о лицензии. 

2. Вводите вашу почту и генерируете ваш HardwareID. После генерации он сразу копируется в буфер обмена, дополнительно копировать его из поля не нужно.
 
3. Шлете мне HardwareID и вашу почту. 
Почтовый адрес: geomobil@mail.ru.
Телеграм: @EgorGeorgiy
Канал с поддержкой скрипта: https://t.me/AGR3D_MKA/566

Я шлю вам ключ и INI файл с ним. 

4. INI файл с ключом GeoAGR.ini кладется в папку со скриптом, после перезапуска скрипт запускается и работает. Если окошко со вводом лицензионного ключа продолжает появляться, свяжитесь со мной, поправлю. 

5. Сохранить настройки интерфейса для последующего старта со скриптом на панели.

6. Перезагрузить 3ds max и удостовериться, что 3ds max не выдает ошибок при старте скрипта. 

7. Работать скрипт с одним ключом может на нескольких 3Д Максах одновременно, для этого папка со скриптом и  INI файлом с ключом копируется в разные 3Д Максы.

8. Обновления ставятся копированием поверх в эту же папку, только INI файл с ключом перезаписывать не надо. Дополнительные шаги из инструкции по установке не нужны. При открытом 3Д Максе достаточно перезаписать папку и заново вызвать скрипт с панели.



Дополнительно. Решение проблем с ImageMagick. 
В INI файле есть переключатель ImageMagickUseScriptDir.
Если переключатель =true, то скрипт будет использовать файлы ImageMagick из папки скрипта,
Если переключатель =false, то скрипт будет использовать файлы ImageMagick из папки по пути, прописанной ниже в ini, например, 
ImageMagick=C:\Program Files\ImageMagick-7.1.2-0-portable-Q16-x64\
или вне защищенного хранилища C:\Program Files\ , например, ImageMagick=E:\ImageMagick\

В случае проблемы с доступом Windows к папке ImageMagick из-за отсутствия прав пользователя, 
отдавать предпочтение пути установки в папку WindowsApps.
